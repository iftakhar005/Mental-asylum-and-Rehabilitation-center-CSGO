🔐 Layer 1: Data Encryption

How It Works:

Step What Happens Example
1. Input Doctor enters patient data "Patient has anxiety and depression"
2. Encryption System encrypts before saving "SGVsbG8gV29ybGQ..." (unreadable)
3. Storage Encrypted data stored in database Database contains encrypted text only
4. Retrieval System checks user role Doctor = decrypt, Receptionist = block
5. Display Show data based on authorization Doctor sees real data, others see [PROTECTED]

Role-Based Access:

✅ Can Decrypt Medical Data

  • Admin
  • Chief Staff
  • Doctor
  • Therapist
  • Nurse

🚫 Cannot Decrypt Medical Data

  • Receptionist
  • Relatives
  • General Staff
  • Parents

🌐 Layer 2: Network Security

6 Security Headers Implemented:

Header Protection Against How It Works
Content-Security-Policy Cross-Site Scripting (XSS) Controls what scripts can run
X-Frame-Options Clickjacking Prevents embedding in frames
X-Content-Type-Options MIME Type Attacks Prevents file type sniffing
X-XSS-Protection XSS Attacks Enables browser XSS filter
Referrer-Policy Information Leakage Controls referrer information
Permissions-Policy Unauthorized Hardware Access Blocks camera/microphone access

Additional Network Security:

  • ✓ Rate Limiting: Max 5 login attempts per minute
  • ✓ HTTPS Enforcement: Automatic redirect in production
  • ✓ Security Logging: All events tracked for audit
